<TagsInput />

Accepts an array of strings and renders them as a tags input.

You must provide onAdd and onRemove callbacks which both receive the tag value as their single attribute. The callbacks must handle updating of the data model (the array of values passed to the data attribute) and re-render the input. The onAdd callback is a good place to clean the entered tag by converting it to lowercase and/or slugifying the value if neccessary.

You may optionally pass a number to the max attribute to limit the number of tags.

Note: Tags passed to the data attribute must be unique. The input simply ignores newly created tags if they already exist. 
<TagsInput data={['technology', 'facebook', 'javascript']} 
           onAdd={this.onTagAddCallback}
           onRemove={this.onTagRemoveCallback}
           max={5} />

<MediaObject />

A simple layout object inspired by Nicole Sullivans media object and Harry Roberts' flag object for placing an image (or other elements/components etc.) next to a block of text. The image can be vertically aligned and the component can be mirrored so the image runs to the right of the text.

Note: Order of children is important. The first child element will be treated as the image and all subsequent children will be grouped as the content.

Default

<MediaObject>
  <img src="example.jpg" alt="example" />
  <p>…</p>
</MediaObject>

Mirrored

<MediaObject mirror={true} >
  <img src="example.jpg" alt="example" />
  <p>…</p>
</MediaObject>

Vertically aligned middle

<MediaObject valign="middle" >
  <img src="example.jpg" alt="example" />
  <p>…</p>
</MediaObject>

Vertically aligned bottom

<MediaObject valign="bottom" >
  <img src="example.jpg" alt="example" />
  <p>…</p>
</MediaObject>

Vertically aligned middle & mirrored

<MediaObject valign="middle" mirror={true} >
  <img src="example.jpg" alt="example" />
  <p>…</p>
</MediaObject>

<FloatingLabelInput />

An input field with a floating <label> which animates after the user begins typing. Inspired by float label pattern.

Note: This component uses Bootstrap 3's input styles. If using in a non-BS3 app you will need to tweak this component's stylesheet accordingly. 
<FloatingLabelInput id="email-input" name="email" label="Email" />

<YoutubeVideo />

Wraps Youtube's embed code in a nicer React interface. All URL parameters are exposed as properties (see the parameter docs for possible options).

The src property accepts the following youtube URL types:

  • https://www.youtube.com/watch?v=R8XAlSp838Y
  • http://www.youtube.com/embed/R8XAlSp838Y

Whichever you provide the URL will be output in the embed style without the protocol.

The protocol can be set explicitly via the protocol property. You would normally only include this for debugging purposes when you are viewing your page from an "offline" protocol (ie. file://).

<YoutubeVideo src="//www.youtube.com/watch?v=R8XAlSp838Y" protocol="http" />

<VimeoVideo />

Like YoutubeVideo, VimeoVideo wraps Vimeo's embed code in a nicer React interface. All URL parameters are exposed as properties (see vimeo's embedding doc for possible options).

The src property accepts the following vimeo URL types:

  • http://vimeo.com/33133076
  • http://player.vimeo.com/video/33133076

Whichever you provide the URL will be output in the embed style without the protocol.

The protocol can be set explicitly via the protocol property. You would normally only include this for debugging purposes when you are viewing your page from an "offline" protocol (ie. file://).

<VimeoVideo src="//vimeo.com/33133076" protocol="http" />
<VimeoVideo src="//player.vimeo.com/video/33133076"
            protocol="http"
            color="ff0000"
            badge="0"
            byline="0"
            portrait="0"
            title="0" />
